## 1. 서비스 개요

### 1.1. 서비스 목적
사용자가 간편하게 여행 일정을 생성하고, 각 날짜별로 세부 계획을 구체적으로 수립하며, Google 지도와 연동하여 장소 정보를 풍부하게 관리할 수 있도록 지원하는 직관적인 모바일 웹 플래닝 서비스 제공을 목적으로 합니다. Tailwind UI Toolkit을 활용한 프리미엄 사용자 경험을 제공합니다.

### 1.2. 핵심 기능 요약
- **여행일정카드 생성**: 여행 국가와 기간을 선택하여 전체 여행의 기본 틀을 자동 생성합니다. (현대적인 카드 UI)
- **여행계획카드 등록**: Day별로 사진, 시간, 장소, 메모 등 상세 계획을 구체적으로 기록합니다. (Badge 기반 유형 표시)
- **Google 지도 API 연동**: 장소 검색 시 Google Place API를 통해 장소의 상세 정보(영업시간, 평점, 웹사이트 등)를 자동으로 불러와 저장합니다.
- **지도 뷰**: 생성된 모든 계획을 지도 위에서 한눈에 확인합니다.

### 1.3. 디자인 시스템 특징
- **Tailwind UI Toolkit**: 프리미엄 컴포넌트 시스템 활용
- **일관된 색상 팔레트**: Blue-600 메인 컬러 기반
- **현대적인 애니메이션**: 부드러운 전환 효과와 호버 상태
- **접근성 고려**: 키보드 네비게이션, 스크린 리더 지원

## 3. 화면 정의: 공통 규칙

### 3.1. 화면 구성 요소
- **Header**: 화면명과 주요 액션 버튼(뒤로가기, 추가, 메뉴 등)을 포함합니다. 고정 헤더로 구현.
- **Content Area**: 각 화면의 주요 콘텐츠가 표시되는 영역입니다. max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 컨테이너 사용.
- **Floating Action Button (필요시)**: 주요 액션을 위한 플로팅 버튼.

### 3.2. 인터랙션 규칙
- **터치(Tap)**: 선택, 실행, 이동 등의 기본 액션을 수행합니다.
- **스와이프(Swipe)**: 목록 삭제, 탭 이동 등의 보조 액션을 수행합니다.
- **롱 프레스(Long Press)**: 수정, 삭제 등 추가 옵션 메뉴를 노출합니다.
- **호버(Hover)**: 데스크탑에서 카드 확대, 그림자 강화 효과.

### 3.3. 공통 디자인 요소
- **컨테이너**: max-w-7xl mx-auto px-4 sm:px-6 lg:px-8
- **카드**: rounded-lg border border-gray-200 bg-white shadow-sm
- **버튼**: h-10 px-4 py-2 rounded-lg transition-colors
- **입력 필드**: h-10 px-3 py-2 rounded-md border border-gray-300

## 4. 메인 화면 (여행일정카드 목록)

### 4.1. 화면 개요
사용자가 생성한 모든 '여행일정카드'가 목록 형태로 노출되는 시작 화면입니다. Tailwind UI 기반의 현대적이고 아름다운 디자인을 적용합니다.

### 4.2. 화면 구성 및 기능 정의

#### Header
- **좌측**: 서비스 로고 (Globe 아이콘 + "Moonwave Travel")
- **우측**: Avatar 기반 사용자 메뉴 (DropdownMenu 컴포넌트)

#### Content Area
- **히어로 섹션**: 
  - 제목: "여행의 모든 순간을 특별하게"
  - 설명: 서비스 소개 텍스트
  - Badge 기반 통계: 국가 수, 계획 수, 여행 수
- **검색 및 필터**:
  - 검색 바: Search 아이콘 포함, 자동완성
  - 뷰 모드 토글: Grid/List 전환 버튼
  - 국가 필터: Badge 스타일 버튼
- **여행일정카드 리스트**:
  - 카드 형태 UI로 각 여행 일정을 표시합니다.
  - 카드 내 표시 정보: 대표 이미지, 여행 국가명 (Badge), 여행 기간, 계획 개수
  - 카드 터치 시: '여행일정 상세 화면'으로 이동합니다.
  - 호버 효과: scale-[1.02], shadow-lg
- **+ 새로운 여행 만들기 버튼**:
  - Blue-600 배경색, Plus 아이콘 포함
  - 카드 형태로 리스트 첫 번째에 배치

#### 초기 화면 (일정 없는 경우)
- 중앙 정렬된 안내 메시지와 "여행 시작하기" 버튼
- Globe 아이콘과 함께 친근한 메시지 표시

#### 통계 섹션 (여행이 있는 경우)
- 3개 카드로 구성: 총 여행, 방문 국가, 총 계획
- 각 카드에 아이콘과 숫자 표시

## 5. 여행일정카드 생성 화면

### 5.1. 화면 개요
새로운 여행의 국가와 기간을 설정하여 '여행일정카드'를 생성하는 화면입니다. 현대적인 폼 UI를 적용합니다.

### 5.2. 화면 구성 및 기능 정의

#### Header
- **좌측**: 뒤로가기 아이콘 (ArrowLeft)
- **중앙**: "새로운 여행 만들기"

#### Content Area
- **히어로 섹션**:
  - Globe 아이콘과 함께 "새로운 여행을 시작해보세요" 메시지
- **여행 대표 이미지 등록**:
  - 드래그 앤 드롭 스타일의 이미지 업로드 영역
  - Camera 아이콘과 안내 텍스트
  - 터치 시: 갤러리 또는 카메라 앱을 실행하여 대표 이미지 선택/촬영
- **여행 정보 입력 폼**:
  - 여행 제목: Input 컴포넌트 사용
  - 여행 국가: MapPin 아이콘 포함, 드롭다운 자동완성
  - 여행 일정: Calendar 아이콘 포함, 사이드 바이 사이드 배치
  - 여행 기간 표시: Blue-50 배경의 정보 카드

#### Footer
- **여행 만들기 버튼**:
  - Blue-600 배경색, Plus 아이콘 포함
  - 로딩 상태: 스피너 애니메이션
  - 비활성화 상태: 회색 처리

## 6. 여행일정 상세 화면 (Day별 목록)

### 6.1. 화면 개요
선택한 여행의 Day별 계획을 확인하고 관리하는 화면입니다. Badge 기반의 Day 탭을 사용합니다.

### 6.2. 화면 구성 및 기능 정의

#### Header
- **좌측**: 뒤로가기 아이콘
- **중앙**: 여행 국가명
- **우측**: 지도 보기 아이콘, 메뉴 아이콘

#### Content Area
- **여행 정보 카드**:
  - 대표 이미지, 제목, 기간 정보
  - Calendar, Clock 아이콘과 함께 메타데이터 표시
- **Day 탭**:
  - Badge 기반 계획 개수 표시
  - 선택된 Day: Blue-600 배경
  - 미선택 Day: White 배경, Gray-200 테두리
- **여행계획카드 리스트**:
  - 시간순 정렬
  - PlanCard 컴포넌트 사용
  - 계획 유형 Badge, 별점, 사진 개수 표시

#### Floating Action Button
- 우하단 고정 위치
- Plus 아이콘, Blue-600 배경
- 선택된 Day에 계획 추가

## 7. 여행계획카드 상세/수정 화면

### 7.1. 화면 개요
여행계획의 상세 정보를 확인하고 수정하는 화면입니다. Badge와 현대적인 폼 요소를 사용합니다.

### 7.2. 화면 구성 및 기능 정의

#### Header
- **좌측**: 뒤로가기 아이콘
- **중앙**: "계획 상세"
- **우측**: 저장 버튼

#### Content Area
- **계획 유형 Badge**: 상단에 계획 유형 표시
- **시간 정보**: Clock 아이콘과 함께 시작/종료 시간
- **장소 정보**:
  - 장소명 (제목)
  - 주소 (MapPin 아이콘)
  - 별점 (SVG Star 아이콘)
- **사진 업로드**: 드래그 앤 드롭 스타일
- **메모**: 텍스트 영역
- **유튜브 링크**: 텍스트 영역

```typescript
  endTime: string;

  youtubeLink?: string;

```
